## 1. 多赛道支持（珠宝/美妆）— 最高价值长期需求

**用户背景：** 用户同时经营珠宝店铺，验证 AI 自媒体效果后计划扩展到珠宝赛道。

**技术挑战：** 当前爬虫、关键词库、prompt 风格全部针对 AI 赛道硬编码。

**调研方向：**

| 方向 | 技术方案 | 参考 |
|------|----------|------|
| 多赛道爬虫 | 爬虫插件化，支持注入赛道关键词列表 | 只需改爬虫配置，无需重构 |
| 赛道化 prompt | 内容生成 prompt 增加「赛道上下文」参数 | 类似 `{{channel}}` 模板变量 |
| 关键词库切换 | 按赛道存储独立关键词白名单/黑名单 | SQLite 表 + 赛道 ID |

**结论：** 技术上可行，核心是数据模型支持赛道 ID。需要新增 `channel` 字段串联爬虫→关键词→内容生成→封面图。

## 2. 定时抓取 + 推送通知

**需求来源：** 每天早上自动抓完热点，打开工具就有当日选题。

**技术方案：**

| 方案 | 实现难度 | 备注 |
|------|----------|------|
| 后端 cron（Python schedule） | 低 | server/main.py 内嵌定时任务，`python3 -m server.main` 常驻进程 |
| macOS launchd | 低 | plist 文件，开机自启，不依赖进程常驻 |
| 云函数（如腾讯云 SCF） | 中 | 支持定时触发，服务器less |
| Railway/Render 托管 | 中 | 只需一个 Web 服务 |

**推荐：** macOS launchd 或后端内嵌 `schedule` 库，最简方案。

**Python 定时库：**
```python
import schedule
def job():
    # 触发爬虫 + 保存到 db
    pass
schedule.every().day.at("07:30").do(job)
```

## 3. AI 改稿建议

**需求来源：** 生成草稿后，用户修改了内容，希望 AI 能对修改后的内容给优化建议。

**技术方案：**
- 后端新增 `/api/review` 端点
- 将用户修改后的正文拼入 prompt：`"请对以下小红书文案给出3条优化建议..."`
- 复用现有的 LLM API（MiniMax/deepseek），无需新增供应商

**参考 prompt 模式：**
```
你是一个专业的小红书内容编辑。请对以下文案给出3条具体优化建议（每条不超过20字）：

---
{draft_body}
---

请直接输出建议：
1. ...
2. ...
3. ...
```
