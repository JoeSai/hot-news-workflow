## 1. 多赛道支持（珠宝/美妆）— 最高价值长期需求

**用户背景：** 用户同时经营珠宝店铺，验证 AI 自媒体效果后计划扩展到珠宝赛道。

**技术挑战：** 当前爬虫、关键词库、prompt 风格全部针对 AI 赛道硬编码。

**调研方向：**

| 方向 | 技术方案 | 参考 |
|------|----------|------|
| 多赛道爬虫 | 爬虫插件化，支持注入赛道关键词列表 | 只需改爬虫配置，无需重构 |
| 赛道化 prompt | 内容生成 prompt 增加「赛道上下文」参数 | 类似 `{{channel}}` 模板变量 |
| 关键词库切换 | 按赛道存储独立关键词白名单/黑名单 | SQLite 表 + 赛道 ID |

**结论：** 技术上可行，核心是数据模型支持赛道 ID。需要新增 `channel` 字段串联爬虫→关键词→内容生成→封面图。

## 2. 定时抓取 + 推送通知

**需求来源：** 每天早上自动抓完热点，打开工具就有当日选题。

**技术方案：**

| 方案 | 实现难度 | 备注 |
|------|----------|------|
| 后端 cron（Python schedule） | 低 | server/main.py 内嵌定时任务，`python3 -m server.main` 常驻进程 |
| macOS launchd | 低 | plist 文件，开机自启，不依赖进程常驻 |
| 云函数（如腾讯云 SCF） | 中 | 支持定时触发，服务器less |
| Railway/Render 托管 | 中 | 只需一个 Web 服务 |

**推荐：** macOS launchd 或后端内嵌 `schedule` 库，最简方案。

**Python 定时库：**
```python
import schedule
def job():
    # 触发爬虫 + 保存到 db
    pass
schedule.every().day.at("07:30").do(job)
```

**需求来源：** 生成草稿后，用户修改了内容，希望 AI 能对修改后的内容给优化建议。

1. ...
2. ...
3. ...

## 5. 多平台草稿扩展调研（微信公众号 / 抖音）

**2026-04-05 探索模式调研**

### 微信公众号

**API 能力**：微信官方提供「新建草稿」API，支持图文内容草稿创建后发布，完全符合项目"只生成草稿"的定位。

**实现流程**：
1. 获取 `access_token`（AppID + AppSecret）
2. 上传封面图到微信素材库（`add_material`）
3. 新建草稿（`draft/add`），传入标题、正文、封面 media_id
4. 发布草稿（用户需在微信后台手动确认发布，或调用 `free_publish`）

**参考项目**：
- `wechat-publisher`（GitHub aibin2026）：Python 实现，Markdown → 微信格式，上传素材+创建草稿
- `we-api`（GitHub shfshanyue）：TypeScript，支持上传时自动替换非微信域名为微信域名

**关键限制**：
- 需要微信公众号账号（订阅号或服务号）
- 需要配置 IP 白名单
- access_token 有效期 2 小时，需自动刷新
- 图文内容需转换为微信的 HTML 格式（CSS 样式受限）

### 抖音

**结论：不适合作为图文草稿扩展目标。**

抖音开放平台主要面向**视频内容**发布（视频上传、管理、数据分析），没有类似微信公众号的图文草稿 API。平台定位与本项目图文种草不匹配。

### 技术建议

若要扩展多平台草稿，**微信公众号是最优先的方向**：
- API 成熟，文档完善
- 草稿模式天然匹配"只生成草稿，用户手动发布"的设计
- 可复用的能力：封面图上传（已有 MiniMax image-01）、Markdown 到微信 HTML 格式转换
- 新增数据模型：`WechatDraft { app_id, access_token, draft_id, status }`
- 新增配置：微信公众号 AppID / AppSecret（可与现有 AI 设置 UI 共用）

## 6. 多赛道扩展调研（珠宝/美妆）

**2026-04-05 探索模式第二轮**

### 用户背景
用户同时经营珠宝店铺，验证 AI 自媒体效果后计划扩展到珠宝赛道。

### 珠宝/美妆内容特点（与 AI 赛道对比）

| 维度 | AI 赛道 | 珠宝/美妆 |
|------|---------|-----------|
| 标题风格 | 技术解析、趋势预测 | 场景代入、情感共鸣 |
| 热门词 | 大模型、GPU、Agent | 宝藏、抗老、胶原、YYDS |
| 内容类型 | 科普、教程、观点 | 种草、测评、探店 |
| 视觉要求 | 封面图+信息图 | 上脸图/上身图（高视觉冲击） |
| 爆款公式 | 数字+对比+时效 | 场景+情感+身份标签 |

### 技术实现路径

**核心改动：赛道上下文注入**

1. **数据模型**：新增 `channel: 'ai' | 'jewelry' | 'beauty'`，贯穿爬虫→关键词→内容生成→封面图全链路
2. **关键词库**：按赛道存储独立白名单/黑名单，SQLite 表 `keyword_channels`
3. **爬虫配置**：每个爬虫可标记支持的赛道（AI 爬虫只出现在 AI 赛道）
4. **Prompt 模板化**：内容生成 prompt 引入 `{{channel_style}}` 变量，切换珠宝/美妆的标题公式和正文风格
5. **封面图**：珠宝强调"光感/质感"，美妆强调"上脸/对比"

### 爆款标题关键词（珠宝/美妆）

**珠宝**：宝藏、YYDS、气质、高级感、绝了、挖到宝、精致、闪瞎、百搭

**美妆**：抗老、胶原、肤感、质地、熬夜、妆前、上脸、YYDS、宝藏、平替

### 结论
技术可行，核心是数据模型增加 `channel` 字段 + prompt 模板变量化。多赛道扩展比多平台扩展更简单，是 v0.20 后最合理的扩展方向。

## 7. 定时抓取调研补充（2026-04-05）

**调研结论：**

| 方案 | 推荐度 | 说明 |
|------|--------|------|
| macOS launchd | ⭐⭐⭐ | 原生方案，可靠性最高，plist 配置后永久生效 |
| 后端 `schedule` 库 | ⭐⭐ | 最简实现，`python3 -m server.main` 常驻进程时直接用，但进程崩则任务停 |
| APScheduler | ⭐⭐ | 功能更全，支持 cron 表达式，适合长期运行的服务 |

**推荐：launchd + 后端 schedule 双保险**
- `schedule` 内嵌到 `server/main.py`，每早 7:30 自动触发爬虫 + 关键词提取，结果写入 DB
- 用户打开工具时当日热点已就绪
- 备用：launchd plist 作为进程保活兜底

**实现要点：**
```python
import schedule
def morning_crawl():
    # 调用爬虫 + 关键词提取，结果持久化到 SQLite
    pass
schedule.every().day.at("07:30").do(morning_crawl)
# 在 uvicorn 启动线程中运行 schedule.run_continuously()
```
