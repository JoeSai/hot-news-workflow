# 产品需求

> 目标用户：AI 赛道小红书自媒体人

## 用户背景

### 核心画像

同时经营**珠宝店铺**和 **AI 科技自媒体号**的运营者。珠宝是主业，AI 自媒体是试验田——先用 AI 赛道验证工具效果（涨粉、效率），成绩满意后再引入珠宝等其他赛道。

### 关键约束

| 约束 | 说明 |
|------|------|
| 不做自动发布 | 平台对 AI 托管发布敏感，工具**只生成草稿**，用户手动发布 |
| 先验证再扩展 | 需要在 AI 号上看到实际增长数据后才会信任并推广到其他赛道 |
| 效果可衡量 | 需要清晰看到「用工具 vs 不用工具」的效率差异和涨粉差异 |

### 当前手动工作流

```
刷各平台找热点 → 筛选 AI 相关选题 → 查资料写文案 → 做封面图 → 手动发布小红书 → 看数据
```

**痛点集中在前三步**：找热点耗时、选题靠直觉、写文案效率低。

---

## 设计红线

1. 绝不自动发布 — 只生成草稿
2. 草稿即参考 — UI 标注为草稿
3. 数据最小化 — 不存储社媒账号
4. 效果可验证 — 保留轻量验证能力，但不把回填和分析放进默认主流程
5. 赛道可迁移 — 模块化支持扩展

## 当前方向

| 方向 | 原则 |
|------|------|
| 抓取先行 | 下一版本优先解决抓取速度、稳定性、AI赛道命中率，再扩展其他功能 |
| 易用度优先 | 默认只服务“抓取 → 选题 → 生成 → 封面/导出”高频链路，减少判断和切换 |
| 聚焦内容产出 | 回填、复杂分析退出默认入口，不再让用户先做低频维护工作 |
| AI 产出多样化 | 先把图文草稿和封面做好，再扩展口播、视频草稿、多形态素材 |
| 进阶能力收纳 | 低频分析能力保留到进阶模式或隐藏入口，不打断主流程 |
